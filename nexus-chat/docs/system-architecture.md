## 3. Docker 服务架构

### 3.1 docker-compose.yml

```yaml
services:
  postgres:
    image: postgres:16-alpine
    container_name: nexus-postgres
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: nexus
      POSTGRES_PASSWORD: nexus123
      POSTGRES_DB: nexus_chat
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./init-db.sql:/docker-entrypoint-initdb.d/init-db.sql:ro

  redis:
    image: redis:7-alpine
    container_name: nexus-redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

  socket-server:
    build: ./socket-server
    container_name: nexus-socket
    ports:
      - "3002:3001"
    environment:
      - DATABASE_URL=postgresql://nexus:nexus123@postgres:5432/nexus_chat
      - CORS_ORIGIN=http://localhost:3000
    depends_on:
      - postgres

volumes:
  postgres_data:
  redis_data:
```

### 3.2 服务端口映射

| 服务 | 容器端口 | 主机端口 | 说明 |
|------|----------|----------|------|
| Next.js | 3000 | 3000 | Web 应用 |
| Socket.io | 3001 | 3002 | 实时消息 |
| PostgreSQL | 5432 | 5432 | 数据库 |
| Redis | 6379 | 6379 | 缓存 |

## 4. 认证流程

### 4.1 邮箱密码登录

```
用户输入邮箱密码
       │
       ▼
┌─────────────────┐
│ NextAuth.js     │
│ CredentialsProv │
│ ider            │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Prisma 查询用户 │
│ (包含 password) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ bcrypt.compare  │
│ 验证密码哈希    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 生成 JWT Token  │
│ 返回 Session    │
└─────────────────┘
```

### 4.2 魔法链接登录

```
用户输入邮箱
       │
       ▼
┌─────────────────┐
│ NextAuth.js     │
│ EmailProvider   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 生成验证 Token  │
│ 存储到数据库    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 发送邮件        │
│ (含验证链接)    │
└────────┬────────┘
         │
         ▼
用户点击链接 → 验证 Token → 创建 Session
```

## 5. 好友请求流程

### 5.1 添加好友流程

```
用户A 添加 用户B 为好友
       │
       ▼
POST /api/friends { friendId: B }
       │
       ▼
┌─────────────────┐
│ FriendshipRepo  │
│ .addFriend()    │
└────────┬────────┘
         │
         ▼
┌─────────────────────────────┐
│ 创建好友请求记录            │
│ { userId: A, friendId: B,   │
│   status: 'pending' }       │
└─────────────────────────────┘
```

### 5.2 接受好友请求流程

```
用户B 接受好友请求
       │
       ▼
GET /api/friends/requests → 获取待处理请求
       │
       ▼
POST /api/friends/accept { friendshipId }
       │
       ▼
┌─────────────────┐
│ FriendshipRepo  │
│ .acceptFriend() │
└────────┬────────┘
         │
         ▼
┌─────────────────────────────┐
│ 1. 更新原请求状态           │
│    status: 'accepted'       │
└────────┬────────────────────┘
         │
         ▼
┌─────────────────────────────┐
│ 2. 创建反向好友关系         │
│    { userId: B, friendId: A,│
│      status: 'accepted' }   │
└─────────────────────────────┘
         │
         ▼
双方互为好友，可在好友列表看到对方
```

### 5.3 拒绝好友请求流程

```
用户B 拒绝好友请求
       │
       ▼
DELETE /api/friends/accept?friendshipId=xxx
       │
       ▼
┌─────────────────┐
│ FriendshipRepo  │
│ .rejectFriend() │
└────────┬────────┘
         │
         ▼
┌─────────────────────────────┐
│ 删除好友请求记录            │
│ (status: 'pending')         │
└─────────────────────────────┘
```

### 5.4 好友关系数据模型

```
┌─────────────────────────────────────────────────────────┐
│                    Friendship 表                         │
├─────────────┬─────────────┬─────────────┬───────────────┤
│ id (PK)     │ userId (FK) │ friendId(FK)│ status        │
├─────────────┼─────────────┼─────────────┼───────────────┤
│ uuid-1      │ A           │ B           │ accepted      │
│ uuid-2      │ B           │ A           │ accepted      │
└─────────────┴─────────────┴─────────────┴───────────────┘

说明：好友关系是双向的，接受请求后创建两条记录
```

## 6. 消息发送流程

### 6.1 同步流程 (HTTP)

```
用户发送消息
       │
       ▼
POST /api/conversations/:id/messages
       │
       ▼
┌─────────────────┐
│ API Route       │
│ 参数校验        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ MessageService  │
│ .sendMessage()  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ MessageRepo     │
│ .create()       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Prisma          │
│ message.create()│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ PostgreSQL      │
│ INSERT INTO     │
└─────────────────┘
```

### 6.2 异步流程 (Socket.io)

```
消息保存成功
       │
       ▼
┌─────────────────┐
│ Socket.io       │
│ .to(room).emit  │
│ ('new-message') │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 所有房间成员    │
│ 收到新消息      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Zustand Store   │
│ 更新消息列表    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ React 组件      │
│ 重新渲染        │
└─────────────────┘
```

## 9. 安全措施

| 安全项 | 实现方式 |
|--------|----------|
| 密码存储 | bcrypt 哈希，10 轮加密 |
| 会话管理 | JWT Token，30 天有效期 |
| CSRF 防护 | NextAuth.js 内置 |
| SQL 注入防护 | Prisma 参数化查询 |
| XSS 防护 | React 自动转义 |
| 传输加密 | TLS 1.3 (生产环境) |

## 10. 监控与日志

### 10.1 日志级别

| 级别 | 用途 |
|------|------|
| error | 错误日志，需要告警 |
| warn | 警告日志，需要关注 |
| info | 信息日志，记录关键操作 |
| debug | 调试日志，开发环境使用 |

### 10.2 关键监控指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| API 响应时间 | < 200ms | P95 |
| 消息送达延迟 | < 500ms | P95 |
| WebSocket 连接数 | < 100,000 | 单实例 |
| 数据库连接池 | < 50 | 并发连接 |
