# demo_codebase 项目结构探索总结

## 项目结构概览

### 核心模块
- **agents/**: 各种AI代理实现，包括 function_call_agent.py、plan_solve_agent.py 等
- **context/**: 上下文构建相关代码
- **core/**: 核心功能模块，包含 agent.py、config.py、llm.py 等
- **memory/**: 记忆系统相关代码，包含 embedding.py、manager.py、rag、storage 等子目录
- **knowledge_base/**: 知识库相关（目前为空）
- **tools/**: 工具集合（目前为空）

### 示例/演示项目
- **demo9/**: 主要的演示项目，包含：
  - `01_context_builder_basic.py` 到 `06_three_day_workflow.py`: 六个演示脚本
  - `codebase_maintainer.py`: 代码库维护器
  - `codebase/`, `data/`, `project/`, `logs/`: 演示项目的子目录结构

### 文档与笔记
- **demo_codebase_notes/**: 项目笔记存储
- **project_notes/**: 项目相关笔记
- **my_flask_app_notes/**: Flask应用相关笔记

### 配置与环境
- **.env**: 环境变量配置文件
- **.profile.ps1**: PowerShell 配置文件

### 技术栈特征
- 基于Python开发
- 包含AI代理、上下文管理、记忆系统等模块
- 采用模块化设计，各功能模块分离
- 使用了RAG（检索增强生成）技术
- 有完整的文档和笔记系统支持开发流程

## 关键发现
1. 项目采用分层架构：核心功能模块 + 代理层 + 应用层
2. 有完善的开发文档和笔记系统
3. demo9 是主要的演示项目，展示了不同功能的使用方式
4. memory 模块是项目的核心组件之一，包含嵌入、管道、存储等多个子模块
5. agents 模块提供了多种不同的AI代理实现，支持不同任务场景
