# 股票买卖信号提醒系统 PRD

**版本**: v1.0
**日期**: 2026-02-25
**状态**: 待评审

---

## 一、Project DNA

> **一句话定义**：为个人投资者提供基于自定义策略的股票买卖信号实时推送服务。

---

## 二、First Action Path

```
用户配置策略参数 → 系统监控实时行情 → 触发条件时微信推送信号
```

---

## 三、Technical Constraints

### 3.1 核心数据模型 (Data Model)

```
┌─────────────────────────────────────────────────────────────┐
│  Strategy (策略)                                             │
│  - id: 策略唯一标识                                          │
│  - stock_code: 股票代码                                      │
│  - stock_name: 股票名称                                      │
│  - strategy_type: 策略类型 (334原则/日线突破/自定义)          │
│  - parameters: 策略参数 (JSON格式)                           │
│  - status: 状态 (active/paused/expired)                     │
│  - created_at: 创建时间                                      │
│  - user_id: 所属用户                                         │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  Signal (信号)                                               │
│  - id: 信号唯一标识                                          │
│  - strategy_id: 关联策略                                     │
│  - signal_type: 信号类型 (buy/sell)                         │
│  - trigger_price: 触发价格                                   │
│  - current_price: 当前价格                                   │
│  - trigger_time: 触发时间                                    │
│  - push_status: 推送状态 (pending/success/failed)           │
│  - message: 信号描述                                         │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  User (用户)                                                 │
│  - id: 用户唯一标识                                          │
│  - wechat_openid: 微信标识 (用于推送)                        │
│  - subscription_status: 订阅状态 (free/paid)                │
│  - created_at: 注册时间                                      │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 状态流转 (State Machine)

```
策略状态机:
  [Created] → [Active] ⇄ [Paused] → [Expired]
                  ↓
              [Deleted]

信号状态机:
  [Generated] → [Pending] → [Pushed] → [Read]
                   ↓
               [Failed] → [Retry] → [Pushed]

推送状态机:
  [Init] → [Sending] → [Success]
                ↓
            [Failed] → [Retry]
```

### 3.3 技术架构约束

| 约束项 | 要求 | 理由 |
|--------|------|------|
| 部署环境 | Windows本地运行 | 用户约束 |
| 数据库 | SQLite (MVP) / PostgreSQL (生产) | 轻量化起步 |
| 行情数据源 | 新浪财经/东方财富/聚合数据 | 免费/低成本 |
| 推送频率限制 | 每策略每分钟最多1次 | 防止刷屏 |

## 四、Feature Spec (审计通过的功能)

### 4.1 策略管理模块

#### 4.1.1 策略创建
- **功能描述**: 用户通过表单配置股票监控策略
- **输入**: 股票代码、策略类型、策略参数
- **输出**: 创建成功的策略记录
- **验收标准**:
  - 支持至少3种预设策略模板（334原则、均线突破、价格阈值）
  - 参数校验：股票代码格式、数值范围合理性
  - 创建后立即进入Active状态开始监控

#### 4.1.2 策略列表
- **功能描述**: 展示用户所有策略及其状态
- **输入**: 用户ID
- **输出**: 策略列表（含状态、触发次数、最后触发时间）
- **验收标准**:
  - 支持按状态筛选
  - 支持暂停/恢复/删除操作

### 4.2 行情监控模块

#### 4.2.1 实时行情获取
- **功能描述**: 定时获取股票实时价格数据
- **触发条件**: 交易时段内，按配置的轮询间隔执行
- **数据字段**: 股票代码、当前价、今开、最高、最低、成交量、涨跌幅
- **验收标准**:
  - 轮询间隔可配置（默认30秒）
  - 失败重试机制（最多3次）
  - 异常数据过滤（如价格为0、涨跌幅超20%）

#### 4.2.2 信号触发判断
- **功能描述**: 根据策略参数判断是否触发买卖信号
- **输入**: 实时行情数据、策略参数
- **输出**: 触发结果（是/否）+ 信号详情
- **验收标准**:
  - 334原则：判断价格是否触及3日/3周/3月均线
  - 均线突破：判断是否突破指定均线
  - 价格阈值：判断是否触及目标价/止损价

### 4.3 微信推送模块

#### 4.3.1 推送发送
- **功能描述**: 将信号通过微信推送给用户
- **输入**: 信号内容、用户微信标识
- **输出**: 推送结果（成功/失败）
- **验收标准**:
  - 推送延迟 < 10秒
  - 失败自动重试（间隔递增：1分钟、5分钟、30分钟）
  - 推送记录持久化存储

#### 4.3.2 推送内容模板
- **功能描述**: 格式化信号内容
- **模板示例**:
  ```
  【买入信号】
  股票：贵州茅台(600519)
  触发价：1850.00
  当前价：1852.50
  策略：334原则-3日均线支撑
  时间：2026-02-25 14:30:00
  建议：价格触及3日均线，可考虑建仓
  ```

### 4.4 信号历史模块

#### 4.4.1 信号记录
- **功能描述**: 存储所有生成的信号记录
- **数据保留**: 最近90天
- **验收标准**:
  - 支持按股票/策略/时间范围查询
  - 支持导出为CSV

## 五、技术路线选型

### 5.1 微信推送方案对比

| 方案 | 实现方式 | 优点 | 缺点 | 推荐度 |
|------|----------|------|------|--------|
| **Server酱** | HTTP API调用微信服务号模板消息 | 稳定、免费额度充足、无需开发 | 依赖第三方服务 | ⭐⭐⭐⭐⭐ |
| **PushPlus** | HTTP API调用微信服务号模板消息 | 功能丰富、支持多种消息类型 | 免费额度较少 | ⭐⭐⭐⭐ |
| **企业微信机器人** | Webhook推送 | 完全免费、稳定 | 需要企业微信账号 | ⭐⭐⭐⭐ |
| **WxPusher** | HTTP API | 免费额度大、支持关注即推送 | 文档较少 | ⭐⭐⭐ |
| **微信个人号机器人** | itchat等库 | 模拟真人发送 | 封号风险极高 | ❌ 不推荐 |

### 5.2 推荐方案

**MVP阶段**: Server酱
- 理由：免费额度每天5条，足够个人使用；接入简单，5分钟完成
- 文档：https://sct.ftqq.com/

**生产阶段**: 企业微信机器人 + Server酱双通道
- 理由：企业微信无限制，Server酱作为备用

### 5.3 行情数据源选型

| 数据源 | 获取方式 | 优点 | 缺点 | 推荐度 |
|--------|----------|------|------|--------|
| **新浪财经** | HTTP接口 | 免费、实时性好 | 无官方文档、可能被限流 | ⭐⭐⭐⭐ |
| **东方财富** | HTTP接口 | 数据全面 | 接口复杂 | ⭐⭐⭐ |
| **Tushare** | Python库 | 数据规范、有文档 | 需要积分、部分接口收费 | ⭐⭐⭐⭐ |
| **AKShare** | Python库 | 开源免费、数据源多 | 部分接口不稳定 | ⭐⭐⭐⭐⭐ |

**推荐方案**: AKShare（主） + 新浪财经（备用）

## 七、风险评估

### 7.1 合规风险 ⚠️ 高优先级

| 风险项 | 风险等级 | 说明 | 应对措施 |
|--------|----------|------|----------|
| 投资咨询资质 | 🔴 高 | 提供买卖建议可能涉及投资咨询业务许可 | 明确定位为"工具"而非"建议"，推送内容加免责声明 |
| 证券投资顾问资格 | 🔴 高 | 收费提供投资建议需要相关资质 | V1阶段免费自用，商业化前咨询法务 |
| 数据版权 | 🟡 中 | 行情数据可能涉及版权 | 使用公开数据源，注明数据来源 |

**免责声明模板**:
```
本系统提供的信号仅供参考，不构成投资建议。
投资有风险，入市需谨慎。
用户应根据自身情况独立做出投资决策。
```

### 7.2 技术风险

| 风险项 | 风险等级 | 说明 | 应对措施 |
|--------|----------|------|----------|
| 数据源不稳定 | 🟡 中 | 第三方数据源可能中断 | 多数据源备份、本地缓存 |
| 推送服务故障 | 🟡 中 | 微信推送服务可能不可用 | 多通道备份、邮件备用 |
| 系统崩溃 | 🟡 中 | 本地服务可能意外停止 | 守护进程、自动重启 |

### 7.3 业务风险

| 风险项 | 风险等级 | 说明 | 应对措施 |
|--------|----------|------|----------|
| 信号准确性 | 🔴 高 | 信号不准确导致用户亏损 | 充分回测、免责声明、逐步开放 |
| 用户信任 | 🟡 中 | 新产品缺乏信任背书 | 先自用验证、积累数据 |
| 竞品压力 | 🟢 低 | 市场已有类似产品 | 聚焦个人策略定制化 |

## 十、附录

### 10.1 334原则说明
- **3日均线**：短线支撑位
- **3周均线**：中线支撑位
- **3月均线**：长线支撑位
- 买入信号：价格触及均线支撑
- 卖出信号：价格跌破均线支撑
